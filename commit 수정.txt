git log함수를 이용하면 
언제 커밋을 했는지 확인할 수 있다.

git log에 있는 해쉬값을 가지고 전 단계로 리셋이 가능하다.

git reset --hard 해쉬코드

--hard를 이용하면 그 시점으로 돌아가고 그 이후에 한 커밋을 다 없애버리겠다는 명령이다.

--soft는 돌아간 이후에도 로그들을 다 남기겠다는 명령이다.

이때
git push -f
의 강제 명령을 사용해서 적용해줄 수 있다.

git commit --amend 

unix의 수정으로 되기 때문에 unix의 명령을 좀 알아야 한다.

푸쉬가 완료 된 상황에서 커밋내용을 수정하고 바뀐 내용을 적용하기 위해서는
git push -f
로 또한 강제 적용 시켜줘야 한다.
